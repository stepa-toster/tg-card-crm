# 🚀 Запуск TG Card CRM Backend

## ЧТО ЭТО ТАКОЕ?

Это Python-сервер который подключается к твоим Telegram аккаунтам.
После запуска ты сможешь писать сообщения прямо с сайта CRM — 
они будут отправляться от имени твоего реального TG аккаунта.

---

## ШАГ 1: Установи Python

Скачай Python 3.11 или новее: https://www.python.org/downloads/

**ВАЖНО при установке на Windows:**
☑️ Поставь галочку "Add Python to PATH"

Проверь что установился:
```
python --version
```
Должно показать: Python 3.11.x (или выше)

---

## ШАГ 2: Получи API ключи Telegram

1. Зайди на сайт: https://my.telegram.org/apps
2. Войди со своим номером телефона
3. Введи код из Telegram
4. Нажми "API development tools"
5. Заполни форму (можно написать что угодно в полях App title и Short name)
6. Нажми "Create application"
7. Скопируй **App api_id** и **App api_hash**

---

## ШАГ 3: Создай файл .env

В папке backend создай файл с именем `.env` (именно с точкой в начале)

Содержимое файла (подставь свои значения):
```
TG_API_ID=12345678
TG_API_HASH=abcdef1234567890abcdef1234567890
PORT=8000
```

---

## ШАГ 4: Установи зависимости

Открой командную строку (cmd) в папке backend:
```
pip install -r requirements.txt
```

Подожди пока всё установится (1-2 минуты).

---

## ШАГ 5: Запусти сервер

```
python main.py
```

Должно появиться:
```
INFO:     Started server process
INFO:     Uvicorn running on http://0.0.0.0:8000
```

---

## ШАГ 6: Добавь свой Telegram аккаунт через CRM

1. Открой CRM: http://localhost:5173
2. Войди: admin / admin123
3. Перейди в ⚙️ Настройки → 📱 Аккаунты
4. Нажми "+ Аккаунт"
5. Введи номер телефона
6. Получишь код в Telegram — введи его
7. Готово! 🎉

---

## ЧАСТЫЕ ПРОБЛЕМЫ

### "python не является внутренней командой"
→ Переустанови Python с галочкой "Add to PATH"
→ Или попробуй: py main.py

### "No module named fastapi"
→ Запусти: pip install -r requirements.txt

### "API_ID не задан"
→ Создай файл .env с твоими ключами (Шаг 3)

### "PhoneCodeInvalidError"
→ Код из Telegram введён неверно, попробуй ещё раз

### Сервер запустился но CRM не видит его
→ Убедись что CRM открыта на http://localhost:5173
→ Убедись что сервер работает на порту 8000
→ Проверь в браузере: http://localhost:8000/api/health

---

## СТРУКТУРА ФАЙЛОВ

```
backend/
  main.py          — основной сервер
  requirements.txt — список зависимостей
  .env             — твои ключи (создай сам!)
  sessions/        — сессии аккаунтов (создаётся автоматически)
    79001234567.session  — сохранённая сессия аккаунта
```

**Сессии важны!** Пока файл .session существует — 
авторизацию делать повторно не нужно. 
При следующем запуске сервера аккаунты подключатся автоматически.

---

## КАК ДОБАВИТЬ НЕСКОЛЬКО АККАУНТОВ?

Просто повтори Шаг 6 для каждого аккаунта.
Каждый аккаунт сохраняется отдельно в папке sessions/.

---

## ОСТАВИТЬ СЕРВЕР РАБОТАТЬ 24/7?

**Бесплатно на Railway.app:**
1. Зарегистрируйся на railway.app
2. New Project → Deploy from GitHub
3. Укажи папку backend
4. Добавь переменные TG_API_ID и TG_API_HASH в Settings → Variables
5. Получи бесплатный URL

**На своём компьютере (Windows):**
Просто не закрывай окно с сервером 😊
