🚂 Railway.app — Пошаговая инструкция

✅ Рекомендуемый вариант: $5 бесплатных кредитов каждый месяц, WebSocket работает, сессии сохраняются постоянно

Шаг 1 — Подготовка GitHub репозитория

1

Создай аккаунт на GitHub

Зайди на github.com → Sign up (бесплатно)

2

Создай новый репозиторий

Нажми + → New repository → Имя: tg-crm-backend → Private → Create

3

Загрузи файлы бэкенда

Открой папку backend/ в проекте. В GitHub нажми "uploading an existing file" и перетащи все файлы из папки backend:

main.py requirements.txt railway.json Procfile runtime.txt

⚠️ НЕ загружай .env и папку sessions/!

Шаг 2 — Получи API ключи Telegram

4

Зайди на my.telegram.org

Открой my.telegram.org/apps

Введи номер своего телефона → получи код в Telegram → введи код

5

Создай приложение

Нажми "API development tools" → заполни форму:

App title: TG Card CRM
Short name: tgcardcrm
Platform: Desktop

Нажми "Create application" → скопируй App api_id и App api_hash

Шаг 3 — Деплой на Railway

6

Зарегистрируйся на Railway

Зайди на railway.app → Login with GitHub

7

Создай новый проект

Нажми "New Project" → "Deploy from GitHub repo" → выбери tg-crm-backend

8

Добавь переменные окружения

В проекте: Variables → New Variable → добавь эти 3 переменные:

ПеременнаяЗначение
TG_API_IDтвой api_id (числа)
TG_API_HASHтвой api_hash (буквы и цифры)
PORT8000
9

Получи публичный URL

В Railway: Settings → Networking → Generate Domain

Получишь URL вида: https://tg-crm-backend.up.railway.app

10

Добавь URL в CRM

В файле src/config.ts замени:

export const BACKEND_URL = "https://tg-crm-backend.up.railway.app";

Или создай .env в корне проекта:

VITE_BACKEND_URL=https://tg-crm-backend.up.railway.app

Шаг 4 — Авторизуй Telegram аккаунт

11

Открой файл add_account.html

Открой backend/add_account.html в браузере → в поле "URL бэкенда" введи свой Railway URL → добавь аккаунт

Или через CRM: Настройки → Аккаунты → Подключить TG

🎉 Готово! Теперь в чате CRM сообщения будут идти с твоего реального Telegram аккаунта

🎨 Render.com — Пошаговая инструкция

⚠️ На бесплатном тарифе сервис засыпает через 15 минут без запросов. При пробуждении сессии восстанавливаются автоматически (файлы .session сохраняются на диске).
1

Загрузи backend на GitHub

Создай репозиторий tg-crm-backend на GitHub, загрузи файлы из папки backend/

2

Зарегистрируйся на Render

render.com → Sign Up with GitHub

3

Создай Web Service

New → Web Service → Connect GitHub → выбери репозиторий

Настройки:

Name: tg-crm-backend
Environment: Python 3
Build Command: pip install -r requirements.txt
Start Command: python main.py
Instance Type: Free
4

Добавь переменные окружения

Environment → Add Environment Variable:

KeyValue
TG_API_IDтвой api_id
TG_API_HASHтвой api_hash
5

Добавь Disk для сохранения сессий

⚠️ ВАЖНО: на бесплатном Render файлы НЕ сохраняются! Нужно добавить Disk:

Disks → Add Disk:

Name: sessions
Mount Path: /opt/render/project/src/sessions
Size: 1 GB
Disk стоит $0.25/GB/мес — это ~7₽ в месяц. Без него при перезапуске нужно будет заново авторизовывать аккаунты.
6

Деплой и настройка URL

Нажми Create Web Service → дождись деплоя (3-5 минут)

Скопируй URL вида https://tg-crm-backend.onrender.com

Вставь его в src/config.ts или .env

☁️ Oracle Cloud — БЕСПЛАТНО НАВСЕГДА

✅ Это единственная платформа с реально бесплатным сервером НАВСЕГДА. 2 CPU, 1GB RAM, без ограничений по времени.
⚠️ Сложнее настроить. Нужна банковская карта для регистрации (деньги НЕ снимаются при выборе Free Tier).
1

Зарегистрируйся на Oracle Cloud

cloud.oracle.com/free → Create Account

Выбери регион ближайший к тебе (Germany Frankfurt или Швейцария)

Введи карту (не снимут деньги при Always Free тарифе)

2

Создай виртуальную машину

Compute → Instances → Create Instance

Name: tg-crm-server
Image: Ubuntu 22.04 (Always Free eligible)
Shape: VM.Standard.E2.1.Micro (Always Free)

Создай или загрузи SSH ключ (сохрани файл .key на компьютере!)

Нажми Create

3

Подключись к серверу

Скопируй Public IP адрес из Oracle → подключись через терминал:

ssh -i путь/к/твоему.key ubuntu@ТУТ_IP_АДРЕС

На Windows скачай PuTTY или используй Windows Terminal

4

Установи Python и зависимости

sudo apt update && sudo apt upgrade -y
sudo apt install python3 python3-pip git -y
mkdir tg-crm && cd tg-crm
5

Загрузи бэкенд

Способ A — через Git:

git clone https://github.com/ТВОЙ_ЛОГИН/tg-crm-backend.git .
pip3 install -r requirements.txt

Способ B — скопируй файлы через SCP:

scp -i твой.key backend/main.py ubuntu@IP:~/tg-crm/
scp -i твой.key backend/requirements.txt ubuntu@IP:~/tg-crm/
6

Создай .env файл

cat > .env << EOF
TG_API_ID=ТУТ_ТВОЙ_API_ID
TG_API_HASH=ТУТ_ТВОЙ_API_HASH
PORT=8000
EOF
7

Открой порт 8000 в Oracle и Ubuntu

В Oracle Console: Virtual Cloud Networks → Security List → Add Ingress Rule:

Source CIDR: 0.0.0.0/0
IP Protocol: TCP
Destination Port Range: 8000

На сервере тоже открой порт:

sudo iptables -A INPUT -p tcp --dport 8000 -j ACCEPT
sudo netfilter-persistent save
8

Запусти как сервис (работает после перезагрузки)

sudo nano /etc/systemd/system/tg-crm.service

Вставь:

[Unit]
Description=TG Card CRM Backend
After=network.target

[Service]
User=ubuntu
WorkingDirectory=/home/ubuntu/tg-crm
ExecStart=/usr/bin/python3 main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable tg-crm
sudo systemctl start tg-crm
9

Используй IP в CRM

Твой бэкенд доступен по адресу: http://ТУТ_IP:8000

Вставь в src/config.ts:

export const BACKEND_URL = "http://123.456.789.0:8000";
🎉 Сервер работает 24/7 бесплатно навсегда!

🪁 Fly.io — Пошаговая инструкция

ℹ️ 3 бесплатных VM, WebSocket работает отлично. Нужна карта для верификации.
1

Установи flyctl

Windows (PowerShell):

iwr https://fly.io/install.ps1 -useb | iex

Mac/Linux:

curl -L https://fly.io/install.sh | sh
2

Зарегистрируйся и войди

fly auth signup
# или если уже есть аккаунт:
fly auth login
3

Создай Dockerfile в папке backend

Создай файл backend/Dockerfile:

FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
RUN mkdir -p sessions
CMD ["python", "main.py"]
4

Задеплой

cd backend
fly launch --name tg-crm-backend --region ams
fly secrets set TG_API_ID=ТУТ_ТВОЙ_ID TG_API_HASH=ТУТ_ТВОЙ_HASH
fly volumes create sessions --size 1
fly deploy
5

Получи URL

fly status
# URL будет вида: https://tg-crm-backend.fly.dev

🏠 Локальный запуск + туннель в интернет

ℹ️ Запускаешь бэкенд на своём компьютере, а через туннель он доступен из интернета (для деплоя CRM на Vercel). Компьютер должен быть включён.
1

Установи Python

Скачай с python.org/downloads → установи (галочка "Add to PATH"!)

2

Создай .env файл

В папке backend/ скопируй .env.example в .env и заполни:

TG_API_ID=12345678
TG_API_HASH=abcdef1234567890abcdef

Ключи: my.telegram.org/apps

3

Запусти бэкенд

В папке backend запусти start.bat (двойной клик) ИЛИ в cmd:

cd backend
pip install -r requirements.txt
python main.py

Бэкенд запустится на http://localhost:8000

4

Вариант A: ngrok (самый простой туннель)

Скачай с ngrok.com → зарегистрируйся → скачай exe

ngrok http 8000

Получишь URL вида: https://abc123.ngrok-free.app

Вставь его в src/config.ts

⚠️ URL меняется при каждом перезапуске ngrok. Платный план ($8/мес) даёт постоянный URL.
5

Вариант B: localtunnel (полностью бесплатно)

npm install -g localtunnel
lt --port 8000 --subdomain tg-crm-backend

URL: https://tg-crm-backend.loca.lt

ℹ️ При первом открытии нужно нажать кнопку подтверждения на странице localtunnel
6

CRM работает локально

Если CRM тоже на localhost, то просто:

export const BACKEND_URL = "http://localhost:8000";

И открывай CRM через npm run dev → http://localhost:5173
